Quiz 2: Programming basics

by Amir Shahmoradi — 05 Apr 2023

This quiz aims at testing your basic knowledge of programming. Donâ€™t forget to push your answers to your remote repository by the end of quiz time. Push all your answers to quiz/2/ folder in your Github project.


  1. (A) What is the closest programming language to machine code (i.e., binary code)?

    (B) Does it need interpretation in order to become machine-comprehensible?

  2. (A) Name the oldest high-level programming language.

    (B) Approximately how many decades is it old? ($\pm15$ years is acceptable answer. the decade it was created is also an acceptable answer)

  3. In what decades C, C++, and MATLAB/Python were created, respectively?

  4. Name an ancestor programming language of C.

  5. Name a programming language ancestor of C++.

  6. Consider the following C++ code. 
    int getSquare(int num) {
    return num * num;
}

int main(){
    return getsquare(2);
}

    What kind of programming error does this code contain? Hint: You can copy and paste this code to https://godbolt.org/ to see the compiler message (by choosing the C++ language from the drop-down menu).

  7. Consider the following Python code. 
    """
Return `True` if the input integer is even.
"""
def isEven(int): return int / 2 == float(int) / 2

    Does this code contain any kind of programming error? If you believe it contains an error, what kind of error is it? Otherwise explain how it yields the correct answer.

  8. Consider the following Fortran code. 
    real :: array(0 : 2)
print *, array(3)
end

    Does this code contain any kind of programming error? Why? Hint: You can copy and paste this code to https://godbolt.org/ to see the Fortran compiler behavior (by choosing the x86-64 gfortran (trunk) compiler from the drop-down menu).

  9. Suppose you write a program that has memory leak. What type of programming error are you dealing with?

  10. What are the order-of-magnitudes of the largest 32-bit and 64-bit IEEE integers?

  11. What are the the smallest and the largest exponents in base 10 that are representable with 32-bit and 64-bit IEEE real/float values?

  12. Suppose you invent a integer binary storage for computers composed of 5-bits. What would be the range of signed integer values in your custom representation?

  13. Consider the IEEE 754 single-precision 32-bit float. How many floats can be represented with such a storage scheme? How many subnormal values do exist in this storage scheme? How many infinite values do exist in this storage scheme? How many NaN values do exist in this storage scheme? What bit pattern would correspond to an infinity? What bit pattern would correspond to a subnormal? What bit pattern would correspond to a NaN?

  14. Why are chaotic phenomena hard to simulate by digital computers?

  15. What is the difference between mutable and immutable objects in computer programming?

  16. Write a recursive function in your favorite programming language that takes a vector of integers as input and outputs the minimum value.

  17. Suppose you implement the matrix multiplication of a large square matrix of size ndim by ndim with a column vector of similar size ndim using the naive definition of matrix multiplication in both C and Fortran. Which version of your code would run faster? C or Fortran? Why?

  18. Suppose you implement the matrix multiplication of a row vector of size ndim with a square matrix of size ndim by ndim using the naive definition of matrix multiplication in both C and Fortran. Which version of your code would run faster? C or Fortran? Why?

  19. Consider the following C programs. Which one would run faster (without any compiler optimizations)? Why?
    1) 
    #include <stdio.h>
#include <time.h>
int matrix[999][999];
void main()
{
    int i, j;
    for (i = 0; i < 999; i++)
        for (j = 0; j < 999; j++)
            matrix[i][j] = matrix[i][j] + (matrix[i][j] * matrix[i][j]);
}

    2)

    #include <stdio.h>
#include <time.h>
int matrix[999][999];
void main()
{
    int i, j;
    for (j = 0; j < 999; j++)
        for (i = 0; i < 999; i++)
            matrix[i][j] = matrix[i][j] + (matrix[i][j] * matrix[i][j]);
}
  20. Consider the following Fortran program. Which loop would run faster in this program (without any compiler optimizations)? Why?
    1) 
    integer :: i, j, matrix(999, 999)
do i = 1, 999
    do j = 1, 999
        matrix(i,j) = matrix(i,j) + (matrix(i,j) * matrix(i,j))
    end do
end do
end

    2)

    integer :: i, j, matrix(999, 999)
do j = 1, 999
    do i = 1, 999
        matrix(i,j) = matrix(i,j) + (matrix(i,j) * matrix(i,j))
    end do
end do
end

  21. Representing matrices as lists in Python can be extremely slow.
    Why are lists unsuitable for numerical storage and fast computation?

  22. What are the three fundamental components of a Turing-complete language?

  23. Suppose we have the following matrix. Can you think of a storage scheme for this matrix that could lead to significant memory saving? 
    my_matrix = [ [ 0 , 0 , 0 , 1 ]
            , [ 2 , 0 , 0 , 0 ]
            , [ 0 , 5 , 0 , 0 ]
            , [ 0 , 0 , 0 , 3 ]
            , [ 0 , 0 , 8 , 0 ]
            ]
